### MOMENT + Chronos-2 联合训练框架设计表
| 模型训练流程细节 | 论文中的依据 / 设计理由 |
| --- | --- |
| **1. 输入数据双轨预处理 (Dual-Track Preprocessing)** |  |
| **MOMENT 端**：对历史数据  使用 MOMENT 原生的 **RevIN** (Reversible Instance Normalization) 进行处理。 | **分布对齐**：MOMENT 的预训练权重是在 RevIN 分布下习得的，保持原生归一化方式能最大化特征提取效率。 |
| **Chronos-2 端**：对全序列（历史+未来）进行标准化（均值 , 标准差 ）后，应用 *$\sinh^{-1}$* 变换。 | **目标空间匹配**：Chronos-2 的 **Quantile Head** 是专门为了预测 $\sinh^{-1}$ 变换后的空间分布而设计的。 |
| **2. 历史特征编码 (History Encoding)** |  |
| **微调策略**：冻结 MOMENT 的 Embedding 层和前 $N-2$  层 Transformer Block，仅**解冻 (Unfreeze) 最后 1-2 层**。 | **领域适应**：保留底层的通用时序特征，同时让高层网络适应特定领域的数据特征。 |
| **3. 特征桥接与重构 (Projection & Reconstruction)** |  |
| **维度对齐**：使用线性层 (Linear/MLP) 将特征维度从$D_{moment}$  映射到 $D_{chronos}$。 | **架构兼容**：建立两个异构模型之间的特征通道。 |
| **未来特征占位**：初始化一组**可学习向量 (Learnable Queries)** 作为未来 Patch 的初始 Embedding。 | **结构填补**：MOMENT 作为编码器无法直接输出未来的 Embedding，需要通过可学习向量填补输入空缺。 |
| **时序信息注入**：显式叠加 **Time Index Embedding** (全局时间索引) 和 **Mask Embedding** (历史/未来标识)。 | **全局时间观**：投影后的特征仅含语义信息，Chronos-2 需要显式的绝对/相对位置信息来理解 $T+H$的整体时间结构。 |
| **序列拼接**：在历史特征和未来特征之间插入特殊的 **REG Token**。 | **序列分隔**：作为分隔符，明确区分历史上下文和待预测的未来区间。 |
| **4. 预测与上下文学习 (Forecasting)** |  |
| **Chronos-2 微调**：在 Attention 层注入 **LoRA** 模块，冻结原模型权重，仅训练 LoRA 参数和 Head。 | **参数高效性**：利用上下文学习能力，在保留预训练知识的同时低成本适配新任务。 |
| **5. 联合优化 (Joint Optimization)** |  |
| **损失函数**：使用 **Quantile Loss (Pinball Loss)** 计算预测分位数与变换后真值  之间的误差。 | **概率预测**：模型输出设计为 21 个分位数，该损失函数是分位数回归的标准优化目标。 |
