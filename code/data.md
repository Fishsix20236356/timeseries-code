#### 1. 流程概述

本模块旨在构建**高保真（High-Fidelity）**的工业时序数据集。与传统统计模型不同，MOMENT 等基础模型依赖原始数据中的微观纹理提取特征。因此，本流程严格限制平滑操作，仅执行必要的数据对齐、物理纠错和完整性补全，输出**物理量纲下**的纯净数据，为后续的双轨归一化（RevIN/Sinh）做好准备。

#### 2. 详细处理步骤

**阶段一：元数据解析与变量筛选 (Metadata Parsing & Variable Selection)**

* **输入源**：`info2.xlsx` (元数据表)
* **处理逻辑**：
1. **目标锁定**：根据关键词（"铁水温度", "铁水Si", "炉渣二元碱度"）锁定预测目标。
2. **协变量锁定**：根据关键词（"喷煤量", "冷水流量", "氧气流量", "风压力"）锁定工艺参数。
3. **映射建立**：建立 `Code -> chnName` 映射，确保后续处理具备可解释性。



**阶段二：时序数据加载与刚性对齐 (Time Series Ingestion & Rigid Alignment)**

* **输入源**：`./json/{code}.json`
* **处理逻辑**：
1. **索引构建**：解析 `time` 字段为标准 DateTime 对象。
2. **宽表合并**：利用 Pandas 索引对齐机制，将所有单变量序列合并为一张宽表。




**阶段三：保守型数据清洗 (Conservative Data Cleaning)**
*本阶段旨在修复数据损坏，同时最大程度保留真实的工况波动（哪怕是剧烈的波动）。*

* **步骤 3.1：多级缺失值填补 (Hierarchical Imputation)**
* **策略**：线性插值 (Linear Interpolation) + 边界填充
* **逻辑**：
1. 对中间段缺失进行**线性插值**，保证时序连续性（基础模型无法处理 NaN），最大连续空隙你自己设定
2. 对首尾残留缺失使用**前向/后向填充 (FFill/BFill)**。


* **关键点**：插值仅用于补全断点，严禁使用复杂的回归插值以免引入人为偏差。


* **步骤 3.2：物理极值截断 (Physical Constraint Verification)**
* **逻辑**：
1. 仅剔除或修正超出物理可能性的**传感器故障值**（如 0, -999, Infinity）。
2. **保留**统计意义上的“异常值”（如突发的高温尖峰），因为这些往往是 MOMENT 需要捕捉的关键故障前兆特征。




**阶段四：数据交付标准 (Hand-off Standard)**

* **输出格式**：Pandas DataFrame 或 Numpy Array。
* **数值状态**：**原始物理量纲 (Raw Physical Scale)**。
* *注意：此时**绝对不要**进行 0-1 归一化或标准化。*
* *原因*：
* **MOMENT 流**：将在模型内部通过 `RevIN` 层动态处理分布变化。
* **Chronos 流**：将在 Dataset `__getitem__` 阶段进行全局均值标准化及  变换。


* **质量指标**：
* 缺失率 (NaN Count) = 0%
* 时间单调递增，无重复时间戳。

### 4. 质量评估指标变更

1. **断点修复图**：展示插值填补了多少空洞。
2. **物理边界图**：展示数据是否都在合理的工艺范围内（如温度未跌破绝对零度或超出炉体耐受极限）。
3. **原始波形图**：确认微小的抖动和尖峰未被误清洗。
