已成功修复 Chronos-2 初始化错误。主要修复内容：

## 问题根源
`Chronos2Model.__init__` 只接受一个参数 `config: Chronos2CoreConfig`，并要求这个 config 对象必须包含 `chronos_config` 属性。原始代码错误地尝试传递两个独立参数：`config` 和 `chronos_config`。

## 修复内容

### 1. 修复 Chronos2Model 初始化（joint_model.py）
**错误代码：**
```python
chronos_core_config = Chronos2CoreConfig(**config.get_chronos_core_config_dict())
chronos_forecasting_config = Chronos2ForecastingConfig(**config.get_chronos_forecasting_config_dict())

self.chronos = Chronos2Model(
    config=chronos_core_config,
    chronos_config=chronos_forecasting_config,  # ❌ 错误
)
```

**修复后：**
```python
# Chronos2Model expects the config object to have a 'chronos_config' attribute
chronos_core_dict = config.get_chronos_core_config_dict()
chronos_core_dict['chronos_config'] = config.get_chronos_forecasting_config_dict()
chronos_core_config = Chronos2CoreConfig(**chronos_core_dict)

self.chronos = Chronos2Model(config=chronos_core_config)  # ✅ 正确
```

### 2. 修复 MOMENT 输出维度问题（joint_model.py）
MOMENT 输出是 4 维张量 `(batch, num_patches, num_channels, hidden_dim)`，需要 squeeze channel 维度：

```python
# Extract patch embeddings
moment_features = moment_output.embeddings
if moment_features.ndim == 4:
    # Squeeze channel dimension (should be 1)
    moment_features = moment_features.squeeze(2)
```

## chronos_config 说明
`chronos_config` 是 `Chronos2ForecastingConfig` 对象，包含：
- context_length: 上下文长度
- patch_size / patch_stride: Patch 配置
- quantiles: 分位数列表（19个分位数）
- use_reg_token: 是否使用 REG token
- use_arcsinh: 是否使用 arcsinh 变换
- time_encoding_scale: 时间编码缩放因子

这些配置用于 Chronos-2 的 patch embedding、instance normalization、输出层和 loss 计算。

   - 将`self.moment.model.parameters()`改为`self.moment.parameters()`

2. **梯度检查点启用代码**（第88-91行）：
